🐜 Hormigasto — Catálogo de componentes UI

Documento de referencia generado a partir del código fuente actual (no modifica la app). Muestra cada patrón visual distinto que existe hoy: botones, cards, badges, inputs, modales, navegación, estados. Es el insumo para construir una guía de estilo formal.

Cómo leer este documento:

Cada ejemplo tiene, debajo o al lado, una etiqueta monospace con el className exacto tal como está en el código fuente:

bg-gray-900 text-white rounded-xl py-3.5

Los hover son reales — pasá el mouse sobre botones/chips para ver el estado. Los disabled se muestran como una segunda instancia real al lado de la normal.

Las cajas con borde ámbar como esta marcan una inconsistencia detectada en el código (no corregida, solo señalada):

⚠ Ejemplo de inconsistencia — así se ven las 6 marcadas a lo largo del documento.

14. Resumen de inconsistencias detectadas

No se corrigieron — quedan documentadas para que la guía de estilo decida cómo unificarlas.

  1. Pending badge con 3 variantes de padding/forma según contexto. ver ↑
  2. Filter chips: activo negro (Etapa/Partida) vs. activo azul (Etiqueta), sin razón semántica clara. ver ↑
  3. Text-link buttons con modificadores de wrapper inconsistentes entre pantallas. ver ↑
  4. Dos escalas de input conviviendo (rounded-xl/text-sm/py-3 vs. rounded-lg/text-xs/py-1.5) sin regla documentada. ver ↑
  5. Color morado de marca solo como hex inline en SystemPromptBox, nunca token Tailwind. ver ↑
  6. Opacidad de disabled inconsistente (opacity-40 / 60 / 30) en componentes similares. ver ↑

15. Archivos fuente (copiados tal cual, sin reescribir)

No existe un archivo de "design tokens" o theme separado en el proyecto. Estos son los únicos dos archivos de estilos/config globales — todo el resto de los colores viven como clases de Tailwind directo en cada componente.

app/globals.css
@tailwind base;
@tailwind components;
@tailwind utilities;

body {
  background-color: #f0f0f0;
  font-family: 'Inter', sans-serif;
  color: #111111;
}

.app-container {
  max-width: 390px;
  margin: 0 auto;
  background: white;
  min-height: 100dvh;
  position: relative;
}
tailwind.config.ts
import type { Config } from "tailwindcss";

const config: Config = {
  content: [
    "./pages/**/*.{js,ts,jsx,tsx,mdx}",
    "./components/**/*.{js,ts,jsx,tsx,mdx}",
    "./app/**/*.{js,ts,jsx,tsx,mdx}",
  ],
  theme: {
    extend: {
      colors: {
        background: "var(--background)",
        foreground: "var(--foreground)",
      },
    },
  },
  plugins: [],
};
export default config;

Nota: colors.background/colors.foreground apuntan a variables CSS --background/--foreground que no están definidas en ningún lado del proyecto (probablemente un resto sin usar del template inicial de Next.js) — el fondo/color real del body está hardcodeado en hex directo en globals.css.
